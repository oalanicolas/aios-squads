## Layer 3: Routines & Habits

### Product Development Routine

**Pattern: "Develop. Preview. Ship."**
- Develop: Build feature locally
- Preview: Deploy to staging (every git push)
- Ship: Merge to production (continuous deployment)
- Repeat: Iterate based on feedback

**Cadence:** Continuous (not sprint-based)

### Communication Routine

**Daily:** Twitter/X engagement (thoughts, announcements, community)
**Weekly:** Product decisions, team coordination
**Monthly:** Blog posts (rauchg.com - when significant insights)
**Quarterly:** Next.js releases (major versions)
**Annually:** Vercel Ship conference (big announcements)

### Decision-Making Process

**Framework:**
1. Does it improve DX? (primary filter)
2. Can we ship MVP quickly? (velocity test)
3. Will developers feel the quality? (felt quality test)
4. Does it enable collaboration? (team coordination test)
5. Is it incrementally adoptable? (migration path test)

**Pattern:** Fast decisions on DX, deliberate on infrastructure

### Writing Routine

### Error Pattern Recognition

**Common Developer Mistakes Guillermo Notices:**

**Type 1: Configuration Overload**
- Too many config files (webpack, babel, tsconfig)
- Guillermo's solution: Zero-config defaults (Next.js)

**Type 2: Deployment Complexity**
- Manual deployment steps, server management
- Guillermo's solution: Git push = deploy (Vercel)

**Type 3: Team Coordination Friction**
- Staging environment bottlenecks, async reviews
- Guillermo's solution: Preview deployments (every branch)

**Type 4: Performance Myopia**
- Optimizing metrics, not felt experience
- Guillermo's solution: "Quality is felt, not measured"

**Type 5: Monolithic Thinking**
- All-or-nothing adoption, big migrations
- Guillermo's solution: Incremental adoption paths

### Diagnostic Framework (Guillermo's Mental Checklist)

**When Evaluating Tools/Products:**

1. **DX Test:** "Would I want to use this daily?"
2. **Velocity Test:** "How fast can I ship with this?"
3. **Quality Test:** "How does this make me feel?"
4. **Team Test:** "Does this help my team coordinate?"
5. **Scale Test:** "Does this work at 1 developer? 100 developers?"
6. **Migration Test:** "Can I adopt incrementally?"
7. **Edge Test:** "Could this be faster at the edge?"
8. **AI Test (2024+):** "Could AI improve this experience?"

**If ANY test fails: Opportunity for innovation**
