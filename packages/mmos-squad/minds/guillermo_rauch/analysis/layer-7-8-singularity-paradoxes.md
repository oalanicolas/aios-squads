**The Tension:**

**Simplicity Pole:**
- Next.js: Zero-config, file-based routing, "just works"
- Vercel: One-click deploy, no server management
- v0: Text prompt → UI (can't be simpler)

**Power Pole:**
- Next.js: Advanced features (ISR, middleware, edge runtime)
- Vercel: Enterprise features (teams, analytics, edge config)
- Full control when needed (custom webpack, advanced routing)

**How Guillermo Holds Both:**

> "Simplicity by default, power when needed"

**Resolution Mechanism:**
- **Default:** Zero-config, opinionated defaults
- **Escape hatch:** `next.config.js` for customization
- **Progressive disclosure:** Simple start, complexity optional

**Manifestation:**
- Next.js: `npx create-next-app` (simple) → custom config (power)
- Vercel: Import from Git (simple) → environment vars, build settings (power)

**Outcome:** Accessible to beginners, powerful for experts
